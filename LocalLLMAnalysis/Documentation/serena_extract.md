# serena_extract.ps1 -- clangd LSP Context Extraction (PowerShell Wrapper)

## Purpose

`serena_extract.ps1` is the PowerShell wrapper that orchestrates clangd-based LSP context
extraction. It validates prerequisites (compile_commands.json, clangd index, Python extract
script), loads configuration from `.env` and presets, builds the command-line arguments, and
invokes `serena_extract.py` via `uv run --python 3.12`.

The extracted LSP context (symbol overviews, incoming references, include dependencies, and
optionally trimmed source) enriches the per-file docs generated by `archgen_local.ps1`. Each source
file gets a `.serena_context.txt` file placed in `LocalLLMAnalysis/.serena_context/`, which Pass 1
loads as additional context when generating architecture docs.

## Prerequisites

| Requirement                   | Details                                                                          |
| ----------------------------- | -------------------------------------------------------------------------------- |
| PowerShell 5.1+ or pwsh 7+    | Uses `Set-StrictMode -Version Latest`                                            |
| `uv` (Python package manager) | Used to run the Python script with `--python 3.12`                               |
| Python 3.12+                  | Via `uv run`                                                                     |
| `clangd`                      | Must be on PATH or specified via `-ClangdPath` (VS2022 Clang components or LLVM) |
| `compile_commands.json`       | At repo root (generate with `generate_compile_commands.py`)                      |
| `.cache/clangd/index/`        | Background index directory (created automatically on first run)                  |
| `serena_extract.py`           | Python extraction script in the same directory                                   |
| `../Common/.env`              | Configuration file                                                               |

## Usage

```powershell
.\serena_extract.ps1 [-TargetDir <dir>] [-Preset <name>] [-Jobs <n>] [-Workers <n>]
                     [-Force] [-SkipRefs] [-Compress] [-MinFreeRAM <GB>]
                     [-RAMPerWorker <GB>] [-EnvFile <path>] [-ClangdPath <path>] [-Test]
```

### CLI Options

| Parameter       | Type   | Default          | Description                                                                            |
| --------------- | ------ | ---------------- | -------------------------------------------------------------------------------------- |
| `-TargetDir`    | string | `"."`            | Subdirectory to scan (relative to repo root)                                           |
| `-Preset`       | string | `""`             | Named preset (`unreal`, `quake`, `generals`, etc.). Controls include/exclude patterns. |
| `-Jobs`         | int    | `2`              | clangd `-j` parallelism per instance (background indexing threads)                     |
| `-Workers`      | int    | `0`              | Max parallel clangd instances. `0` = auto-detect based on free RAM.                    |
| `-Force`        | switch | off              | Re-extract all files even if context already exists                                    |
| `-SkipRefs`     | switch | off              | Skip reference queries (much faster, symbols only -- no incoming references)           |
| `-Compress`     | switch | off              | Compress LSP output: collapse class methods, show top symbols by count                 |
| `-MinFreeRAM`   | double | `6.0`            | Minimum free RAM in GB to maintain during extraction                                   |
| `-RAMPerWorker` | double | `5.0`            | Estimated RAM per clangd instance in GB (for auto-scaling)                             |
| `-EnvFile`      | string | `../Common/.env` | Alternative `.env` file                                                                |
| `-ClangdPath`   | string | `"clangd"`       | Path to clangd binary                                                                  |
| `-Test`         | switch | off              | Run the built-in unit test suite (tests wrapper functions only, no clangd needed)      |

## How It Is Invoked

**Standalone:**
```powershell
cd C:\Coding\Generals
.\LocalLLMAnalysis\serena_extract.ps1 -Preset generals
```

**Via ArchPipeline.py (analysis mode):**
Called during the optional setup phase before the main analysis steps:
```
python Common/ArchPipeline.py analysis
```

## Input Files

| Input                   | Location                       | Description                                   |
| ----------------------- | ------------------------------ | --------------------------------------------- |
| Source files            | `<repo>/<TargetDir>/`          | C/C++ files matching include/exclude patterns |
| `compile_commands.json` | `<repo>/compile_commands.json` | Clang compilation database                    |
| `.env`                  | `../Common/.env`               | Configuration for preset and patterns         |

## Output Files

| Output        | Location                                                    | Description          |
| ------------- | ----------------------------------------------------------- | -------------------- |
| Context files | `LocalLLMAnalysis/.serena_context/<rel>.serena_context.txt` | Per-file LSP context |

## Environment Variables / .env Keys

| Key                  | Default          | Description                                     |
| -------------------- | ---------------- | ----------------------------------------------- |
| `PRESET`             | `""`             | Named preset (overridden by `-Preset` CLI flag) |
| `INCLUDE_EXT_REGEX`  | Preset-dependent | File extension include regex                    |
| `EXCLUDE_DIRS_REGEX` | Preset-dependent | Directory exclude regex                         |

## Supported Presets

| Preset names                                  | Include pattern                                                  | Key exclusions                                                                                                           |
| --------------------------------------------- | ---------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------ |
| `quake`, `quake2`, `quake3`, `doom`, `idtech` | `.c`, `.cc`, `.cpp`, `.cxx`, `.h`, `.hh`, `.hpp`, `.inl`, `.inc` | `.git`, `architecture`, `build`, `out`, `dist`, `obj`, `bin`, `baseq2`, `baseq3`, `base`                                 |
| `unreal`, `ue4`, `ue5`                        | `.cpp`, `.h`, `.hpp`, `.cc`, `.cxx`, `.inl`                      | `.git`, `architecture`, `Binaries`, `Build`, `DerivedDataCache`, `Intermediate`, `Saved`, `ThirdParty`, `GeneratedFiles` |
| `generals`, `cnc`, `sage`                     | `.cpp`, `.h`, `.hpp`, `.c`, `.cc`, `.cxx`, `.inl`, `.inc`        | `.git`, `architecture`, `LocalLLMAnalysis`, `Dep`, `Debug`, `Release`                                                    |
| (empty/default)                               | `.c`, `.cc`, `.cpp`, `.cxx`, `.h`, `.hh`, `.hpp`, `.inl`, `.inc` | `.git`, `architecture`, `build`, `out`, `dist`, `obj`, `bin`                                                             |

## Exit Codes

| Code     | Meaning                                                                         |
| -------- | ------------------------------------------------------------------------------- |
| `0`      | Success                                                                         |
| `2`      | Missing `compile_commands.json`, missing `serena_extract.py`, or unknown preset |
| Non-zero | Propagated from `serena_extract.py` on failure                                  |

## Examples

**Example 1: Extract LSP context for an Unreal Engine project**
```powershell
cd C:\Coding\UnrealEngine
.\LocalLLMAnalysis\serena_extract.ps1 -Preset unreal -Jobs 4
```

**Example 2: Fast extraction (symbols only, skip references)**
```powershell
.\serena_extract.ps1 -Preset generals -SkipRefs
```
Runs much faster by skipping cross-file reference queries. The output will contain symbol
overviews and includes but no "Incoming References" section.

**Example 3: Force re-extraction for a specific subdirectory**
```powershell
.\serena_extract.ps1 -TargetDir Engine/Source/Runtime/Core -Preset unreal -Force
```
Re-extracts all files under `Engine/Source/Runtime/Core/` even if context files already exist.
