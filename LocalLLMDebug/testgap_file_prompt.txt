Focus on what is MISSING from the tests, not what is present.

Output this exact markdown structure:

# <source file path>

## Overall
GOOD | PARTIAL | POOR — one sentence justifying the rating.

## Untested Public Interface
For each public function, method, or class in the source file not exercised by any test:

## Untested Error Paths

## Broken or Misleading Tests
For each test that appears to assert the wrong thing, use incorrect data, or has a comment that contradicts the actual fixture values:

1. [HIGH/MEDIUM/LOW] what to test and why
